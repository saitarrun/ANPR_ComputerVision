# ANPR Development Environment

Complete local development setup that mirrors production infrastructure.

## Overview

The ANPR project uses **Docker Compose** to create an isolated, production-like environment on your local machine. This includes:

- **PostgreSQL 16** — same as production
- **Redis 7** — same as production
- **MinIO** — S3-compatible storage for development
- **FastAPI** — ANPR backend API with hot-reload
- **Celery** — background task worker
- **pgAdmin** — database UI (optional)
- **Redis Commander** — Redis inspection UI (optional)

All services are defined in `docker-compose.yml` with development overrides in `docker-compose.override.yml`.

## Quick Start

### 1. Clone & Setup (First Time Only)
```bash
git clone <repo-url>
cd ComputerVision\ Project

# Copy env template and generate secrets
cp .env.local.example .env.local

# Generate missing secrets
python -c "import secrets; print('JWT_SECRET=' + secrets.token_urlsafe(32))" >> .env.local
python -c "from cryptography.fernet import Fernet; print('FERNET_KEY=' + Fernet.generate_key().decode())" >> .env.local
python -c "from cryptography.fernet import Fernet; print('CELERY_ENCRYPTION_KEY=' + Fernet.generate_key().decode())" >> .env.local
```

### 2. Start Services (5 seconds)
```bash
docker-compose up -d --wait
```

### 3. Initialize Database
```bash
# Install Python dependencies
python -m pip install -e .

# Run migrations + seed test data
python scripts/init_dev_env.py
```

### 4. Verify Setup
```bash
# Run tests
python -m pytest tests/unit -q

# Or check health
curl http://localhost:8000/healthz
```

**Done!** Your environment is ready.

## Usage

### View API Documentation
```bash
open http://localhost:8000/docs
```

### Run Tests
```bash
# All tests
python -m pytest tests/ -q

# Watch mode (reruns on changes)
python -m pytest tests/ -q --tb=short -k "not slow" --looponfail

# With coverage
python -m pytest tests/ --cov=api --cov=db --cov-report=html
```

### Inspect Database
```bash
# pgAdmin (web UI)
open http://localhost:5050

# or PostgreSQL CLI
docker-compose exec postgres psql -U anpr -d anpr
```

### Inspect Redis
```bash
# Redis Commander (web UI)
open http://localhost:8081

# or Redis CLI
docker-compose exec redis redis-cli
```

### Follow Logs
```bash
# All services
docker-compose logs -f --tail=50

# Specific service
docker-compose logs -f api --tail=50
docker-compose logs -f celery-worker --tail=50
```

### Stop Services
```bash
docker-compose down
```

## Environment Variables

### Required in `.env.local`
These are generated by `make dev-setup`:
- `JWT_SECRET` (min 32 chars)
- `FERNET_KEY` (44 chars, base64)
- `CELERY_ENCRYPTION_KEY` (44 chars, base64)

### Auto-Set by docker-compose
- `DATABASE_URL=postgresql+asyncpg://anpr:dev_password@postgres:5432/anpr`
- `REDIS_URL=redis://redis:6379/0`
- `S3_ENDPOINT_URL=http://minio:9000`

### Override for Testing
Edit `.env.local`:
```bash
ANPR_ENV=dev
LOG_LEVEL=DEBUG
FRONTEND_ORIGINS=http://localhost:3000,http://localhost:5173,http://localhost:5050
```

## Common Tasks

### Reset Database (Destructive)
```bash
# Drop all tables and recreate
docker-compose down -v
docker-compose up -d
python scripts/init_dev_env.py --reset
```

### Hot-Reload Not Working?
```bash
# Restart container
docker-compose restart api

# Or check logs
docker-compose logs api | grep -i error
```

### Port Already in Use?
```bash
# Find conflicting process
lsof -i :5432   # PostgreSQL
lsof -i :6379   # Redis
lsof -i :8000   # API

# Kill it
kill -9 <PID>
```

### Out of Disk Space?
```bash
# Check Docker usage
docker system df

# Clean up unused resources
docker system prune -a

# Or just this project
docker-compose down -v
```

## Troubleshooting

### Services Won't Start
```bash
# Check logs
docker-compose logs

# Rebuild images
docker-compose build --no-cache

# Reset everything
docker-compose down -v
docker-compose up --build
```

### Database Connection Refused
```bash
# Wait for PostgreSQL
docker-compose exec postgres pg_isready -U anpr

# Check logs
docker-compose logs postgres

# Verify environment
docker-compose exec api env | grep DATABASE_URL
```

### Worker Not Processing Tasks
```bash
# Check worker is running
docker-compose ps celery-worker

# View logs
docker-compose logs celery-worker

# Check Redis
docker-compose exec redis redis-cli KEYS '*'

# Restart worker
docker-compose restart celery-worker
```

### Tests Fail with "Connection Refused"
```bash
# Ensure docker-compose is running
docker-compose ps

# Initialize database if needed
python scripts/init_dev_env.py

# Run tests again
python -m pytest tests/integration -q
```

## Advanced

### Debug with pdb
```bash
# Add breakpoint in code
import pdb; pdb.set_trace()

# Run with Docker
docker-compose exec api python -m pdb my_script.py

# View logs to see breakpoint
docker-compose logs api
```

### Profile Code Performance
```bash
# Add profiling
from cProfile import Profile
profiler = Profile()
profiler.enable()
# ... code ...
profiler.disable()
profiler.print_stats()
```

### Custom Docker Image
```bash
# Modify Dockerfile, rebuild
docker-compose build --no-cache api

# Restart
docker-compose restart api
```

### Inspect Container Filesystem
```bash
docker-compose exec api bash
# ls -la
# grep DATABASE_URL
# exit
```

## Support

**Need help?**

1. Check [DOCKER_TROUBLESHOOTING.md](DOCKER_TROUBLESHOOTING.md) for common issues
2. Review `docker-compose logs` for error details
3. Verify `.env.local` has all required secrets
4. Try a clean reset: `docker-compose down -v && docker-compose up --build`

**Still stuck?** Check:
- `/var/lib/docker/containers/` for container logs
- `docker system events` for real-time Docker activity
- `docker inspect <container>` for container details
